Design a beautiful browser game play screen for a programming puzzle game with a modern Nintendo-like puzzle aesthetic.

Important:
- Do not copy any specific Nintendo IP, characters, worlds, or branding
- Capture the feeling of a premium polished puzzle game: playful, tactile, highly readable, charming, spatial, and inviting
- This must feel like a real game, not a dashboard, coding tool, or educational worksheet

## Overall visual style
Create a bright, premium, toy-like puzzle world with:
- soft rounded forms
- chunky readable shapes
- clean materials
- subtle depth and shadows
- playful but controlled color palette
- polished animation
- strong visual clarity

The mood should feel:
- cheerful
- clever
- tactile
- magical-tech
- cozy but smart
- premium and game-like

Avoid:
- flat admin UI
- enterprise panels
- dark hacker aesthetics
- overly realistic sci-fi
- generic educational app look
- excessive text or tiny controls

## Scene composition
The main play screen should focus on a floating puzzle board in the center stage.
The board should feel like a handcrafted miniature world suspended in space, like a toy diorama.

Layout hierarchy:
1. Main hero area - floating puzzle board
2. Secondary play area - tactile command programming strip
3. Small supporting HUD - level info, playback controls, status

Do not let the command area overpower the board.
The board is the star.

## Board design
Create an isometric or pseudo-isometric tile board made of chunky 3D blocks.
Each tile should feel physical and toy-like, with rounded edges or softened corners.
The world should look handcrafted and premium.

Board elements:
- neutral walkable tiles
- glowing target tiles
- gaps / void spaces
- visible height differences
- edge depth and soft shadows
- subtle ambient particles or atmosphere if it helps

The board should feel like a clean puzzle object placed on a stage.
Use depth, lighting, and materials to make it feel satisfying.

Visual qualities:
- readable geometry
- clear elevation
- soft directional light
- pleasant shadowing
- subtle glow for active puzzle elements
- no clutter

## Robot character
Create a small charming robot that feels like a premium puzzle companion.
It should be simple, iconic, and expressive through pose and motion rather than facial detail.

Robot design goals:
- compact silhouette
- oversized head or defining top shape for readability
- clear facing direction
- glowing eyes, visor, or chest light
- smooth rounded forms
- toy-like finish
- friendly and clever personality

Animation behavior:
- subtle idle bob or breathing motion
- clean turn animation
- satisfying hop or step movement
- tiny anticipation before moving
- celebratory motion on success

The robot should feel lovable and functional, not like a mascot slapped onto a UI.

## Command programming area
The command composer should feel like a physical interaction surface.
The player should feel like they are placing chunky puzzle command pieces into sockets.

Include:
- MAIN routine
- PROC1 routine
- PROC2 routine
- command palette

Visual style:
- chunky square or rounded-rectangle command tiles
- bold pictograms
- tactile pressed / hover states
- clear active playback highlight
- soft material shading
- satisfying snap-to-slot feel

Commands should look like toy control pieces, not code blocks or form inputs.

## Playback controls
Add game-like controls for:
- Run
- Step
- Pause
- Reset

Style:
- large
- icon-first
- polished
- tactile
- easy to hit
- clearly interactive

These should feel like part of the toy puzzle console, not browser buttons.

## HUD and feedback
Keep the HUD minimal and elegant.

Include:
- level title
- current world / puzzle number
- success or failure feedback
- optional stars / progress markers

Feedback style:
- compact
- clean
- animated
- satisfying
- never modal-heavy unless it is a polished end-of-level moment

Failure should be visible but not harsh.
Success should feel rewarding with light, motion, and charm.

## Colors and materials
Use a premium family-friendly color palette.
Suggested direction:
- warm neutrals for base tiles
- bright accent colors for active targets
- a few saturated UI accents for commands
- soft sky / gradient / dreamy background
- one clear robot accent glow color

Materials should feel:
- smooth
- polished
- toy-like
- slightly glossy or satin
- not plastic overload, not hyper-real

## Motion
Use animation generously but with discipline.
Everything should feel alive and responsive.

Animate:
- robot idle
- robot move / rotate
- target activation
- active command playback
- slot placement
- button presses
- success feedback
- gentle board ambiance

Do not make the screen noisy.
Motion must support clarity.

## Technical direction
Use React + TypeScript.
Prefer HTML Canvas for the board if it helps create a richer game feel.
Use Framer Motion for interface transitions if useful.
The result should be componentized and production-quality.

## Deliverable
Create a single polished playable mock screen for one sample level using mocked data.
The output should include:
- the full play screen
- floating isometric puzzle board
- charming robot
- tactile command strips
- playback controls
- status feedback
- enough animation to feel like a real game

## Non-negotiable
If it starts looking like a dashboard, redesign it.
If it starts looking like a generic coding app, redesign it.
It must feel like a premium puzzle game with modern Nintendo-like readability, charm, tactility, and polish.
